vscode eslint 配置

最近开始学习 React，配置了下 vscode 的 eslint 和格式化

我的 es6 格式：

  • 结尾加分号
  • 2个空格缩进
  • 用单引号

依赖

npm

  • eslint
  • babel-eslint

使用 cnpm 或者 npm config set registry https://registry.npm.taobao.org 可以加快 npm install 速度

vscode 插件

  • eslint
  • Prettier - JavaScript formatter
  • EditorConfig for VS Code (用来控制不同语言的文件的格式）

配置

配置 eslint 解析器、规则

在项目根目录创建 .eslintrc.json 文件

配置 eslint 忽略的文件

有些文件不需要使用到 eslint 减少编辑器报错

在项目根目录创建 .eslintignore

配置 Prettier

在项目根目录创建 .prettierrc

配置 EditorConfig

配置不同语言文件的不同缩进格式

在 HOME 目录下，创建文件。vim ~/.editorconfig 

配置 vscode

编辑 vscode 的配置文件

保存自动格式化
